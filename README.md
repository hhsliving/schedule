# 가전팀 편성표 뷰어

주차별 TV 편성표를 사업부 / 팀편성 두 가지로 넘겨보는 정적 뷰어입니다.
엑셀 RAW 파일을 매크로와 동일한 서식으로 가공한 뒤 이미지로 렌더해서 쌓아둡니다.

**보기 →** https://hhsliving.github.io/schedule/

---

## 사용법

| 조작 | 설명 |
|---|---|
| **사업부 / 팀편성** 탭 | 시트 전환 (키보드 `1` / `2`) |
| **‹ ›** 버튼 | 주 이동 (키보드 `←` / `→`) |
| **찾기** 드롭다운 | 연도 · 월 · 주차 선택해서 바로 이동 |
| **하단 주차 레일** | 클릭해서 바로 이동. 주황 표식 = 월 교차 주간 |
| **이미지 클릭** | 원본 크기로 확대 (드래그로 이동, 다시 클릭하거나 `ESC` 로 축소) |

기본 상태에서는 편성표 전체가 스크롤 없이 화면에 맞춰집니다.

---

## 구조

```
index.html                    뷰어 (단일 파일 — CSS/JS 내장)
weeks/                        편성표·비중 원본 (다 섞여 있어도 됨)
  ...편성표....xlsx             편성표 (A1 제목으로 주차 자동 인식)
  undefined_....xlsx           비중 (같은 커밋의 편성표를 따라감)
  .pairs.json                  편성표-비중 짝 기록 (자동 생성)
images/                       주차별 편성표 이미지
  dept-{연도}-{월}-{주차}.png     사업부 시트
  team-{연도}-{월}-{주차}.png     팀편성 시트
tools/                        이미지 생성 스크립트
.github/workflows/build.yml   자동화 (raw/ 변경 시 실행)
```

주차 목록은 `index.html` 상단 `WEEKS` 배열에 들어 있습니다.

```js
const WEEKS = [
  { y: 2026, m: 7, w: 4, span: "07/20 ~ 07/26", cross: false },
  { y: 2026, m: 7, w: 5, span: "07/27 ~ 08/02", cross: true  }
];
```

- `cross` : 월이 바뀌는 주간이면 `true`
- 정렬은 자동이며, 가장 최근 주차가 첫 화면으로 열립니다

## 주차 추가하기

`weeks/` 폴더에 **편성표와 비중 엑셀을 그냥 다 넣으면** 됩니다. 폴더를 나눌 필요 없습니다.

```
weeks/
    ...편성표....xlsx              ← 편성표 (파일명 그대로)
    undefined_20260716.xlsx        ← 비중 (파일명 그대로)
    ...다른 주차 편성표....xlsx
    ...
```

**딱 한 가지 규칙 : 편성표와 비중을 같은 번에 올린다 (같은 커밋).**
GitHub 웹에서 Upload files 할 때 그 주차의 편성표 + 비중을 **한꺼번에** 끌어다 놓고
커밋하면 됩니다. 그러면 시스템이 "이 편성표와 이 비중은 한 세트"로 묶습니다.

- **파일 이름은 아무래도 좋습니다.** 내용을 보고 편성표/비중을 구분하고,
  주차는 편성표 A1 제목에서 읽습니다.
- 비중이 어느 주차 건지는 **같은 커밋의 편성표**를 따라갑니다. 그래서 폴더도, 이름도 필요 없습니다.
- 한 번 짝지어지면 기록되므로(`weeks/.pairs.json`), 나중에 다른 파일을 올려도 유지됩니다.

푸시하면 Actions 가 이미지 생성 → `index.html` 갱신 → 자동 커밋까지 합니다.

> 그래야 "같은 세트"로 인식됩니다. 편성표는 이미 있어도 다시 add 하면 됩니다.

## 편성 비중 채우기

편성표와 **같은 폴더**에 넣기만 하면 됩니다. 이름은 아무래도 좋습니다.
리포트에서 **B열 '팀'이 가전/리빙/주방인 행**을 찾아 값을 가져오므로,
숫자를 직접 옮겨 적을 필요가 없습니다.

**일반 주간 — 리포트 1개**

| 리포트 | → 표의 |
|---|---|
| D열 현재주차 | **주** |
| F열 월누적 | **월** |

**월 교차 주간 — 리포트 2개**

브라우저에서 비중을 두 번 받으면 이름이 이렇게 됩니다:

```
undefined_20260716.xlsx        ← 원본 (먼저) = 지난달 월 비중
undefined_20260716 (1).xlsx    ← 사본 (1)   = 새 달 1주차 비중
```

이 **`(1)` 접미사로 순서를 판단**합니다. 이름을 바꿀 필요 없습니다.
먼저 받은 지난달 것이 원본(접미사 없음), 나중 받은 새 달 것이 `(1)` 이 되도록만
받으시면 됩니다.

| 리포트 | → 표의 |
|---|---|
| **원본** (접미사 없음) 의 F열 월누적 | **"OO월 누적"** (지난달 누적) |
| **`(1)`** 사본의 D열 현재주차 | **"△△월 1주차"** (새 달 1주차) |

> 예) 07월 5주(07/27~08/02) → 1번째 = 7월 누적, 2번째 = 8월 1주차

개수가 안 맞으면 경고를 띄우고 최선으로 처리합니다.
월교차인데 1개만 있으면 일반 주간처럼, 일반 주간인데 2개면 첫 번째만 씁니다.


**비중 파일이 없는 주차는 비중표가 공란**으로 만들어집니다.
나중에 채워도 되고, 잘못 넣었으면 고쳐서 다시 커밋하면 그만입니다.

## 이미지 생성 (tools/)

GitHub Actions 가 자동으로 돌리므로 보통은 직접 실행할 일이 없습니다.
로컬에서 돌리려면 아래가 필요합니다.

```bash
sudo apt-get install libreoffice-calc poppler-utils fonts-noto-cjk fonts-crosextra-carlito
pip install openpyxl pillow
python3 tools/batch.py
```

`fonts-crosextra-carlito` 는 **반드시** 필요합니다. Calibri 메트릭 호환 폰트로,
열 너비 픽셀 환산의 기준이라 없으면 렌더 폭이 달라집니다.

`batch.py` 가 하는 일:

1. RAW 파일 A1 제목(`◆ 2026년 07월 4주 편성표 (07/20 ~ 07/26)`)에서
   연도 · 월 · 주차 · 기간 · 월교차 여부를 자동 판독
2. `build_sheet.py` 로 매크로와 동일한 서식 적용
   (요일 그룹 테두리, 가전/리빙/주방/Hit&Best 색칠, 비중표 삽입)
3. `render.py` 로 인쇄 설정 후 LibreOffice → PDF → PNG 렌더
4. 뷰어용 `WEEKS` 배열 출력

비중을 채우려면 `batch.py` 의 `RATIOS` 에 해당 주차를 추가하고,
없으면 비워둔 채로 생성됩니다.

## 알아둘 점

- **양식 통일** : 01~06월 원본은 열 너비가 7월보다 좁습니다.
  가장 최근 주차의 편성표를 기준으로 열 너비를 맞춰 렌더합니다.
- **Hit & Best** : 07월 RAW 부터 있는 항목이라 과거 주차에는 표시되지 않습니다.
- **기본 스타일 폰트** : openpyxl 이 저장 시 기본 스타일의 폰트 참조를 바꿔버려
  같은 열 너비라도 렌더 폭이 달라집니다(Arial 9 → 1546px / Calibri 12 → 1869px).
  `build_sheet.py` 가 저장 후 Calibri 12 로 고정해 전 주차 1869px 로 통일합니다.
